# 문서

문서는 시스템의 기본적인 콘텐츠 구성 요소입니다. 생성, 업데이트, 관리할 수 있는 개별 콘텐츠를 나타냅니다.

## 주요 기능

- **버전 관리**: 변경 사항 추적 및 이력 유지
- **메타데이터 지원**: 문서에 대한 추가 정보 저장
- **콘텐츠 유형**: 다양한 콘텐츠 형식 지원
- **관계**: 다른 엔티티와 문서 연결

## 사용법

```typescript
// 문서 구조 예시
interface Document {
  id: string;
  title: string;
  content: string;
  metadata: Record<string, any>;
  createdAt: Date;
  updatedAt: Date;
}
```

## 모범 사례

1. 의미 있는 메타데이터 포함
2. 적절한 콘텐츠 유형 사용
3. 올바른 관계 유지
4. 버전 관리 가이드라인 준수

## 문서 생성하기

문서를 생성하는 가장 간단한 방법은 `cms:document` 아티즌 명령어를 사용하는 것입니다:

```bash {{ title: '문서 생성 명령어' }}
php artisan cms:document Page
```

이 명령어는 다음과 같은 파일들을 자동으로 생성합니다:

- `app/Documents/Page.php`: 문서 클래스
- `app/Orchid/Resources/PageResource.php`: Orchid 리소스
- `database/migrations/[timestamp]_create_pages_table.php`: 마이그레이션 파일

## 문서의 특징

CMS-Orbit의 문서는 다음과 같은 강력한 기능을 제공합니다:

1. **버전 관리**: 문서의 모든 변경 사항을 추적하고 이전 버전으로 되돌릴 수 있습니다.
2. **상태 관리**: 초안, 검토, 게시 등 다양한 상태를 관리할 수 있습니다.
3. **다국어 지원**: 여러 언어로 콘텐츠를 작성하고 관리할 수 있습니다.
4. **SEO 최적화**: 메타데이터, URL 슬러그, 카노니컬 URL 등을 관리할 수 있습니다.

## 문서 예제

다음은 페이지 문서의 예제입니다:

```php {{ title: '페이지 문서 예제' }}
<?php

namespace App\Documents;

use Illuminate\Database\Eloquent\Model;
use Orchid\Attachment\Attachable;
use Orchid\Filters\Filterable;
use Orchid\Screen\AsSource;

class Page extends Model
{
    use AsSource, Filterable, Attachable;

    protected $fillable = [
        'title',
        'slug',
        'content',
        'meta_title',
        'meta_description',
        'status',
        'published_at',
    ];

    protected $casts = [
        'status' => 'string',
        'published_at' => 'datetime',
    ];

    public function scopePublished($query)
    {
        return $query->where('status', 'published')
            ->where('published_at', '<=', now());
    }
}
```

## 문서의 사용 사례

문서는 다음과 같은 상황에서 유용합니다:

1. **정적 페이지**: 회사 소개, 연락처 등 정적 콘텐츠 관리
2. **블로그 포스트**: 블로그 콘텐츠 관리
3. **뉴스 기사**: 뉴스 콘텐츠 관리
4. **상품 설명**: 상품 상세 정보 관리
