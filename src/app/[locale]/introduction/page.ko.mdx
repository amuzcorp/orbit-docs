# CMS-Orbit 소개

CMS-Orbit은 라라벨 기반의 현대적인 콘텐츠 관리 시스템입니다. {{ className: 'lead' }}

## CMS-Orbit이란?

CMS-Orbit은 웹사이트와 애플리케이션을 위한 강력한 콘텐츠 관리 솔루션을 제공합니다.

### 목적과 특징

- **모듈식 아키텍처**: 확장 가능하고 유연한 구조
- **다중 테넌시**: 여러 사이트를 하나의 인스턴스로 관리
- **다국어 지원**: 글로벌 웹사이트 구축을 위한 완벽한 지원
- **관리자 패널**: 직관적인 Orchid 기반 관리 인터페이스

### 아키텍처

CMS-Orbit은 다음과 같은 계층 구조를 가집니다:

1. **프레젠테이션 계층**
   - 사용자 인터페이스
   - API 엔드포인트
   - 관리자 패널

2. **비즈니스 계층**
   - 서비스 로직
   - 비즈니스 규칙
   - 워크플로우

3. **데이터 계층**
   - 데이터베이스
   - 캐시
   - 파일 시스템

### 주요 구성 요소

- **모델**: 데이터 구조와 관계 정의
- **엔티티**: 비즈니스 로직 캡슐화
- **문서**: 콘텐츠 관리의 핵심
- **테마**: 프론트엔드 디자인 시스템
- **API**: 외부 시스템과의 통합

## 주요 기능

### 콘텐츠 관리 시스템

- **문서 관리**: 버전 관리, 상태 관리, 다국어 지원
- **미디어 관리**: 이미지, 비디오, 문서 등의 자산 관리
- **SEO 최적화**: 메타데이터, URL 구조, 사이트맵 관리

### 다중 테넌시 지원

- **사이트 격리**: 데이터베이스, 파일 시스템, 세션 격리
- **리소스 공유**: 테마, 플러그인, 설정 공유
- **중앙 관리**: 모든 사이트의 중앙 집중식 관리

### 다국어 지원

- **언어 관리**: 다국어 콘텐츠 관리
- **번역 시스템**: 자동/수동 번역 지원
- **로케일 설정**: 지역별 설정 관리

### 관리자 패널

- **Orchid 플랫폼**: 직관적인 관리 인터페이스
- **사용자 관리**: 역할 기반 접근 제어
- **대시보드**: 실시간 모니터링 및 분석

### API 지원

- **RESTful API**: 표준화된 API 엔드포인트
- **인증/인가**: OAuth2, 토큰 기반 인증
- **문서화**: Swagger/OpenAPI 지원

## 시스템 요구사항

### 서버 요구사항

- **웹 서버**: Apache/Nginx
- **PHP**: 8.3 이상
- **데이터베이스**: MySQL 8.0 이상, PostgreSQL 13 이상
- **메모리**: 최소 2GB RAM
- **스토리지**: 최소 10GB 디스크 공간

### PHP 요구사항

- **확장 프로그램**:
  - PDO
  - OpenSSL
  - Mbstring
  - Tokenizer
  - XML
  - Ctype
  - JSON
  - BCMath
  - Fileinfo

### 데이터베이스 요구사항

- **MySQL 8.0 이상**
  - InnoDB 스토리지 엔진
  - UTF-8 문자셋
  - 최소 1GB 스토리지

- **PostgreSQL 13 이상**
  - UTF-8 문자셋
  - 최소 1GB 스토리지

### 확장 프로그램 요구사항

- **Composer**: 패키지 관리
- **Node.js**: 프론트엔드 빌드
- **Git**: 버전 관리
