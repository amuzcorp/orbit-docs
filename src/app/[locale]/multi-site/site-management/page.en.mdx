# Site Management

This guide explains how to manage sites.

## Site Management Features

- **Site Listing**: View and manage all sites
- **Site Status**: Monitor site status
- **User Management**: Manage site-specific users
- **Content Management**: Manage site-specific content
- **Resource Management**: Manage site-specific resources

## Site Management Methods

### 1. Site Listing

```php {{ title: 'Site Listing' }}
use Orbit\Tenancy\Tenant;

// List all sites
$tenants = Tenant::all();

// List only active sites
$activeTenants = Tenant::where('status', 'active')->get();

// List sites with specific conditions
$filteredTenants = Tenant::where('created_at', '>=', now()->subDays(30))->get();
```

### 2. Site Status Management

```php {{ title: 'Site Status Management' }}
use Orbit\Tenancy\Tenant;

$tenant = Tenant::find('site-a');

// Check site status
$status = $tenant->status();

// Activate site
$tenant->activate();

// Deactivate site
$tenant->deactivate();

// Suspend site
$tenant->suspend();
```

### 3. User Management

```php {{ title: 'User Management' }}
use Orbit\Tenancy\Tenant;

$tenant = Tenant::find('site-a');

// List users
$users = $tenant->users()->get();

// Create user
$user = $tenant->users()->create([
    'name' => 'New User',
    'email' => 'user@site-a.example.com',
    'password' => Hash::make('password'),
]);

// Manage user permissions
$tenant->permissions()->assignRole($user, 'editor');
```

### 4. Content Management

```php {{ title: 'Content Management' }}
use Orbit\Tenancy\Tenant;

$tenant = Tenant::find('site-a');

// List contents
$contents = $tenant->contents()->get();

// Create content
$content = $tenant->contents()->create([
    'title' => 'New Content',
    'body' => 'Content body',
    'status' => 'published',
]);

// Update content
$content->update([
    'status' => 'draft',
]);
```

## Site Management Commands

```bash {{ title: 'Site Management Commands' }}
# List sites
php artisan tenancy:list

# Check site status
php artisan tenancy:status site-a

# Activate site
php artisan tenancy:activate site-a

# Deactivate site
php artisan tenancy:deactivate site-a
```

## Best Practices

1. Regularly monitor site status
2. Strictly manage user permissions
3. Regularly backup content
4. Monitor resource usage
5. Regularly review security settings
