# 자주 묻는 질문

CMS-Orbit의 멀티사이트 기능에 대해 자주 묻는 질문과 답변을 알아봅니다. {{ className: 'lead' }}

## 일반 질문

### Q: CMS-Orbit이란 무엇인가요?

A: CMS-Orbit은 Laravel 기반의 멀티사이트 CMS(Content Management System)입니다. 하나의 코드베이스로 여러 사이트를 관리할 수 있으며, 각 사이트는 독립적인 데이터베이스와 파일 시스템을 가집니다.

### Q: 멀티사이트 기능의 장점은 무엇인가요?

A: 멀티사이트 기능의 주요 장점은 다음과 같습니다:
- 하나의 코드베이스로 여러 사이트 관리
- 각 사이트의 독립적인 데이터 보장
- 리소스 공유를 통한 효율적인 운영
- 중앙 집중식 관리 및 모니터링
- 확장성 있는 아키텍처

### Q: 어떤 종류의 사이트에 적합한가요?

A: CMS-Orbit은 다음과 같은 사이트에 적합합니다:
- 기업 웹사이트
- 이커머스 사이트
- 블로그 및 뉴스 사이트
- 교육 사이트
- 커뮤니티 사이트

## 기술 질문

### Q: 데이터베이스는 어떻게 관리되나요?

A: 각 사이트는 독립적인 데이터베이스를 가지며, 다음과 같이 관리됩니다:
```php
// 데이터베이스 설정
'connections' => [
    'tenant' => [
        'driver' => 'mysql',
        'database' => 'tenant_' . $tenant->id,
        // ... 기타 설정
    ],
],
```

### Q: 파일 시스템은 어떻게 관리되나요?

A: 각 사이트는 독립적인 파일 시스템을 가지며, 다음과 같이 관리됩니다:
```php
// 파일 시스템 설정
'filesystem' => [
    'disk' => 'local',
    'root_override' => [
        'local' => '%storage_path%/app/tenants/%tenant_id%',
        'public' => '%storage_path%/app/public/tenants/%tenant_id%',
    ],
],
```

### Q: 세션은 어떻게 관리되나요?

A: 각 사이트는 독립적인 세션을 가지며, 다음과 같이 관리됩니다:
```php
// 세션 설정
'session' => [
    'driver' => 'redis',
    'connection' => 'tenant',
    'cookie' => 'tenant_session',
],
```

## 운영 질문

### Q: 사이트를 어떻게 백업하나요?

A: 사이트는 다음과 같이 백업할 수 있습니다:
```bash
# 데이터베이스 백업
php artisan tenancy:backup-database site-a

# 파일 백업
php artisan tenancy:backup-files site-a

# 설정 백업
php artisan tenancy:backup-settings site-a
```

### Q: 사이트를 어떻게 복원하나요?

A: 사이트는 다음과 같이 복원할 수 있습니다:
```bash
# 데이터베이스 복원
php artisan tenancy:restore-database site-a backup-id

# 파일 복원
php artisan tenancy:restore-files site-a backup-id

# 설정 복원
php artisan tenancy:restore-settings site-a backup-id
```

### Q: 사이트를 어떻게 모니터링하나요?

A: 사이트는 다음과 같이 모니터링할 수 있습니다:
```bash
# 성능 모니터링
php artisan tenancy:monitor-performance site-a

# 오류 모니터링
php artisan tenancy:monitor-errors site-a

# 사용자 모니터링
php artisan tenancy:monitor-users site-a
```

### Q: 사이트를 어떻게 확장하나요?

A: 사이트는 다음과 같이 확장할 수 있습니다:
```bash
# 수평 확장
php artisan tenancy:scale-horizontal site-a

# 수직 확장
php artisan tenancy:scale-vertical site-a

# 로드 밸런서 설정
php artisan tenancy:load-balancer site-a
```

### Q: 사이트를 어떻게 통합하나요?

A: 사이트는 다음과 같이 통합할 수 있습니다:
```bash
# API 통합
php artisan tenancy:api-integrate site-a

# 데이터 통합
php artisan tenancy:data-integrate site-a site-b

# 서비스 통합
php artisan tenancy:service-integrate site-a
```

### Q: 사이트를 어떻게 유지보수하나요?

A: 사이트는 다음과 같이 유지보수할 수 있습니다:
```bash
# 시스템 업데이트
php artisan tenancy:update site-a

# 패키지 업데이트
php artisan tenancy:update-packages site-a

# 유지보수 모드
php artisan tenancy:maintenance site-a --enable
```

### Q: 문제가 발생하면 어떻게 해결하나요?

A: 문제는 다음과 같이 해결할 수 있습니다:
```bash
# 상태 확인
php artisan tenancy:status site-a

# 성능 진단
php artisan tenancy:diagnose site-a

# 데이터베이스 복구
php artisan tenancy:repair-database site-a

# 파일 시스템 복구
php artisan tenancy:repair-files site-a
```
