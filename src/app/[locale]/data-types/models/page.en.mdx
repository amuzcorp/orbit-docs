# Model Data Types

Model data types define the structure and behavior of your database models. They help ensure data consistency and provide type safety for your database operations.

## Key Features

- **Type Definition**: Define the structure of your models
- **Validation Rules**: Ensure data integrity
- **Database Schema**: Define database table structure
- **Custom Types**: Create custom model types

## Example

```typescript
// Example model type definition
interface ModelType {
  id: string;
  name: string;
  table: string;
  columns: {
    id: {
      type: 'bigIncrements';
      primary: true;
    };
    name: {
      type: 'string';
      length: 100;
      nullable: false;
    };
    description: {
      type: 'text';
      nullable: true;
    };
    status: {
      type: 'enum';
      values: ['active', 'inactive', 'pending'];
      default: 'pending';
    };
    created_at: {
      type: 'timestamp';
      nullable: false;
    };
    updated_at: {
      type: 'timestamp';
      nullable: false;
    };
  };
  indexes: {
    name_index: {
      columns: ['name'];
      unique: true;
    };
  };
}
```

## Best Practices

2. Implement proper validation
