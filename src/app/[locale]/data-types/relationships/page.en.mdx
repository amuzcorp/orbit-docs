# Relationship Data Types

Relationship data types define how different entities are connected to each other. They help establish and maintain relationships between your data models.

## Key Features

- **Relationship Types**: Define different types of relationships
- **Cardinality**: Specify one-to-one, one-to-many, or many-to-many relationships
- **Constraints**: Define relationship constraints
- **Cascading**: Configure cascading behavior

## Example

```typescript
// Example relationship type definition
interface RelationshipType {
  id: string;
  name: string;
  type: 'belongsTo' | 'hasOne' | 'hasMany' | 'belongsToMany';
  source: {
    entity: string;
    foreignKey: string;
  };
  target: {
    entity: string;
    localKey: string;
  };
  constraints: {
    onDelete: 'cascade' | 'setNull' | 'restrict';
    onUpdate: 'cascade' | 'setNull' | 'restrict';
  };
  pivot?: {
    table: string;
    columns: string[];
  };
}
```

## Best Practices

1. Use meaningful relationship names
2. Define appropriate constraints
3. Consider performance implications
4. Document relationship definitions
